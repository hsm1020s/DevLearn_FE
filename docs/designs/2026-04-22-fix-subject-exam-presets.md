## 목표
SUBJECT_CATALOG의 SQLP·DAP·정보관리기술사 모의고사 프리셋(문항수·시간)과 과목 분류가 실제 시험 규격과 달라 정정한다. 사용자가 공식 자료(KDATA/Q-Net)를 근거로 확인해 준 값을 그대로 반영한다.

### 정확한 규격 (근거: 사용자 제공)

- **SQLP** — 총 72문항(객관식 70 + 실기 2), **180분**, 100점. 3과목 구조:
  1. 데이터 모델링의 이해(10문항)
  2. SQL 기본 및 활용(20문항)
  3. SQL 고급 활용 및 튜닝(40문항)
- **DAP** — 총 76문항(객관식 75 + 실기 1), **240분(4시간)**, 100점. 2024 개편 이후 6과목:
  1. 전사아키텍처 이해(10)
  2. 데이터 요건 분석(10)
  3. 데이터 표준화(10)
  4. 데이터 모델링(25)
  5. 데이터베이스 설계와 이용(10)
  6. 데이터 품질 관리 이해(10)
- **정보관리기술사 필기** — 4교시제, 교시당 100분(총 400분), 전면 논술형.
  - 1교시: 13문제 중 10문제 선택(단답/약술)
  - 2~4교시: 교시별 6문제 중 4문제 선택(논술)
  - 합격: 400점 중 240점(60%) 이상, 과락 없음

### 이번 태스크 스코프
- `SUBJECT_CATALOG`의 `examPreset` 3개 정정
- `checklist` 과목 분류 정확화 (SQLP 3과목, DAP 6과목)
- `description` 을 실제 규격 한줄 요약으로 교체

### 비(非)목표
- **정보관리기술사는 본질적으로 논술형**이라 객관식 mock과 어긋난다는 사용자 지적은 별도 태스크로 분리한다. 이번엔 앱의 기존 객관식 mock 구조를 그대로 유지하고, 프리셋은 **1교시(단답형) 기준**(10문항·100분)으로 근사한다. 2~4교시 논술 지원·모범답안 키워드 매칭은 후속 설계 대상.
- 합격 기준(과목별 40% 과락 등) UI 노출은 이번에 추가하지 않음. 프리셋 배너에 간단히 문구 보완만.
