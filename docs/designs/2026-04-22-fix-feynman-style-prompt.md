## 목표
파인만/한줄요약 학습 스타일이 실제 백엔드 연결 상태(VITE_MOCK_API=false)에서 **전혀 동작하지 않는** 버그를 수정한다. 현재 프론트가 메시지 앞에 `[파인만 모드] ` 라는 한글 태그만 붙여 보내는데, 백엔드 LLM은 이 태그를 해석하는 로직이 없어 일반 응답을 돌려준다. mock에서만 동작하던 반쪽짜리 구현이었다.

### 근본 문제
- `src/hooks/useStreamingChat.js:17-21` 의 `STYLE_PREFIX = { feynman: '[파인만 모드] ', summary: '[한줄요약] ' }`는 **mock의 detectStyle이 알아보는 내부 토큰**일 뿐, LLM에게는 의미 없는 한글 태그.
- `.env.local` → `VITE_MOCK_API=false` 이므로 실제 백엔드로 요청이 가고, 백엔드는 태그를 무시해 일반 응답만 반환 → 사용자가 체감하는 "파인만 기능 동작 안 함".

### 해결 방향 (Option A — FE 단독)
프리픽스를 **LLM이 자연어로 이해 가능한 시스템 지시문**으로 교체. 백엔드가 따로 스타일 필드를 처리하지 않아도 일반 LLM 호출만으로 스타일 효과를 얻는다.

```js
const STYLE_PROMPT = {
  feynman:
    '[학습 모드 지시문] 아래 사용자 메시지에 대해 "파인만 기법"으로 응답해줘. ' +
    '즉, 사용자가 내용을 스스로 설명한 경우 빠진 개념·잘못 이해한 부분을 짚어주고, ' +
    '개념 설명을 요청한 경우 아이 수준의 비유로 단계별로 풀어 설명해줘.\n\n' +
    '사용자 메시지:\n',
  summary:
    '[학습 모드 지시문] 아래 사용자 메시지에 대해 "한 문장 요약" 스타일로 응답해줘. ' +
    '핵심만 남긴 한 문장을 먼저 주고, 그 뒤에 근거를 불릿 2~3줄로 덧붙여줘.\n\n' +
    '사용자 메시지:\n',
};
```

- **LLM 호환성**: 자연어 지시라 GPT/Claude/Gemini/Llama 계열 모두 그대로 해석 가능.
- **Mock 호환성**: `detectStyle`은 문자열 시작 매칭이므로 새 프리픽스 앞단 토큰(`[학습 모드 지시문]`)으로 바꾸면 그대로 감지.
- **뱃지/UI**: 사용자 메시지 UI에는 여전히 원본 content만 보이고, 프리픽스는 `doStream → streamMessage` 단계에서만 합쳐 서버로 전달(현행 구조 유지).

### 부가 버그 수정
- `useStreamingChat.js:187` 의 409 재시도 분기에서 사용자 메시지를 다시 추가할 때 `meta: { style }`이 빠져 있어 재시도 메시지에만 뱃지가 사라지는 엣지 케이스. `handleSend`의 최초 `addMessage` 와 동일한 meta를 전달하도록 수정.

### 비(非)목표
- 백엔드 API에 `style` 필드 추가. 본 FE 스코프 밖.
- 추가 스타일(예: "면접관 모드") 도입. 향후 태스크.
- 토큰 사용량 최적화(시스템 프롬프트 중복 등). 지시문은 짧게 유지.

- mock 동작 유지
