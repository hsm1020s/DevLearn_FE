## 목표
카카오톡 채팅방 투명도 슬라이더처럼, **모든 화면**의 패널(사이드바/카드/채팅 패널/모달 등) 배경 알파값을 사용자가 슬라이더로 조절해 "선명 ↔ 흐림" 시각 효과를 낸다. 크림/베이지 톤 정체성은 유지하므로 배경 이미지는 도입하지 않고, body 베이스(`--color-bg-primary`) 위 surface 레이어(`--color-bg-secondary` / `--color-bg-tertiary` / 보더)의 알파값만 0.4 ~ 1.0 범위에서 조절한다. 동일 크림 계열이지만 톤 단계 차이(#FAF9F5 → #F1EFE7 → #E8E5DB)가 있어 알파가 낮아지면 패널이 body에 녹아들며 흐려지고, 1.0이면 또렷이 분리된다.

## 변경 범위

**스타일 토큰 (globals.css)**
- `:root` 및 `[data-theme="dark"]` 에 RGB triplet 변수 추가 (`--rgb-bg-secondary`, `--rgb-bg-tertiary`, `--rgb-border-light`, `--rgb-border-medium`).
- `--ui-clarity: 1.0` 전역 변수 추가 (slider가 이 값을 직접 갱신).
- 기존 `--color-bg-secondary` / `--color-bg-tertiary` / `--color-border-light` / `--color-border-medium` 정의를 `rgb(var(--rgb-...) / var(--ui-clarity))` 로 교체. body 베이스인 `--color-bg-primary` 와 텍스트/액센트 토큰은 알파 미적용(항상 1.0).

**상태 (useAppStore.js)**
- `uiClarity: 1.0` state + `setUiClarity(v)` action 추가 (0.4 ~ 1.0 클램프).
- `partialize` 에 `uiClarity` 포함 → localStorage 영속화.

**적용 효과 훅 (App.jsx 또는 신규 hook)**
- `useEffect` 에서 `useAppStore` 의 `uiClarity` 변화를 구독해 `document.documentElement.style.setProperty('--ui-clarity', value)` 로 root 스타일 갱신. 초기 마운트 시도 동기화.

**UI: 슬라이더 (Sidebar.jsx 푸터)**
- 사이드바 펼침 상태일 때 "기능개선 제안" 위쪽 또는 "문서 업로드" 아래쪽에 한 줄짜리 슬라이더 행 추가:
  - 좌측 아이콘 (`Sun` / `Eye` 등 lucide), 라벨 "화면 선명도", 우측에 `<input type="range" min="40" max="100" step="5">` (값은 0.4 ~ 1.0 매핑).
  - collapsed 사이드바에서는 노출하지 않는다 (아이콘만 표시할 공간 부족 → 우선순위 낮음).
  - 모바일 사이드바 열림 시에도 동일하게 노출.

**부착 위치 결정 근거**
- 글로벌 설정이지만 별도 SettingsModal을 신설하면 단일 슬라이더 하나만으로 과한 구조가 된다. Sidebar 푸터는 이미 "문서 파이프라인 / 문서 업로드 / 기능개선 제안 / 사용자" 등 글로벌 액션이 모여있어 자연스럽다.
- `userMenu` 드롭다운(설정/로그아웃) 안에는 넣지 않는다. 슬라이더는 즉시 시각 피드백이 필요해서 드롭다운이 닫혔다 열렸다 하면 UX가 끊긴다.

## 구현 계획

1. **globals.css 토큰 분리**
   - `:root` 와 `[data-theme="dark"]` 양쪽에 `--rgb-bg-secondary`, `--rgb-bg-tertiary`, `--rgb-border-light`, `--rgb-border-medium`, `--ui-clarity: 1.0` 추가.
   - 기존 `--color-bg-secondary` 등 4개 토큰을 `rgb(... / var(--ui-clarity))` 로 재정의. body 배경, 텍스트, primary/success/warning/danger 등은 변경하지 않는다.
   - `--ui-clarity` 변경 시 즉시 반영되는지 DevTools 로 수동 확인 (Step 2 단위 테스트 첫 단계).

2. **useAppStore 확장**
   - `uiClarity: 1.0` 초기값 + `setUiClarity` action 추가. clamp 0.4 ~ 1.0.
   - `partialize` 에 `uiClarity` 추가.

3. **App.jsx 동기화 훅**
   - `App.jsx` 최상단에서 `const uiClarity = useAppStore((s) => s.uiClarity);` 구독 + `useEffect(() => { document.documentElement.style.setProperty('--ui-clarity', String(uiClarity)); }, [uiClarity]);`.

4. **Sidebar 슬라이더 UI**
   - `Sidebar.jsx` 의 footer 영역(`border-t border-border-light px-3 py-2 flex flex-col gap-0.5`) 첫 번째 자식으로 슬라이더 행 추가.
   - 슬라이더는 native `<input type="range">` 사용 (커스텀 컴포넌트는 추후 필요시 분리). accent 색은 primary.
   - JSDoc / 한국어 인라인 주석 (왜 0.4 가 하한인지: 그 이하는 panel 가독성이 무너짐).

5. **다크 모드 확인**
   - `[data-theme="dark"]` 에서도 RGB triplet 정의 누락 없는지 확인. (현재 다크 토글 UI는 없지만 토큰 일관성 유지.)
