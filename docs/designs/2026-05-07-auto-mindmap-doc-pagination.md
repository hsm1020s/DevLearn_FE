## 목표
마인드맵 패널의 "자동 생성" 탭 — 문서(책) 리스트가 길어져 스크롤만으로는 탐색이 불편한 문제 해결.
페이지당 10개씩 잘라 보여주고 번호형 페이지네이션(이전 · 1 2 3 … · 다음)으로 이동할 수 있게 한다.
챕터 리스트(문서 선택 후 화면)는 이번 범위에서 제외 — 문서 리스트만 페이징한다.

## 변경 범위
- 수정 파일: `src/components/mindmap/AutoMindmapTab.jsx` (단일 파일)
  - 문서 리스트 렌더 영역(현재 `docs.map(...)`)을 페이지 슬라이스 + 컨트롤로 교체
- API/스토어/백엔드 변경 없음 — 클라이언트 사이드 페이징
- 새 컴포넌트 추가 없음 — 동일 파일 내 인라인 처리(범위가 작음, 재사용 수요 불명확)

## 구현 계획
1. `AutoMindmapTab.jsx` 상단 상태에 `page`(1-indexed) 추가, `PAGE_SIZE = 10` 상수 정의.
2. 파생값 계산:
   - `totalPages = max(1, ceil(docs.length / PAGE_SIZE))`
   - `pagedDocs = docs.slice((page-1)*PAGE_SIZE, page*PAGE_SIZE)`
3. `docs.length`가 줄어들어 현재 page가 범위를 벗어나면 1페이지로 리셋(useEffect로 보정).
4. 문서 리스트 `map` 대상을 `pagedDocs`로 교체.
5. 리스트 하단에 페이지네이션 컨트롤 렌더:
   - `docs.length > PAGE_SIZE`일 때만 노출(10개 이하면 컨트롤 자체를 숨김)
   - 구성: `‹ 이전` · 페이지 번호 버튼들 · `다음 ›`
   - 페이지 번호는 현재 페이지 ±2 윈도우 + 양 끝(1, last) + `…` 생략 표기 (총 페이지 1~7개일 땐 전부 노출)
   - 현재 페이지 버튼은 `bg-primary text-white`, 그 외는 hover 스타일
   - 이전/다음 버튼은 끝단에서 `disabled` + opacity 다운
6. 스타일링은 기존 토큰(`bg-bg-secondary`, `text-text-tertiary`, `text-primary` 등)으로만 처리 — 하드코딩 색 금지.
7. `selectedDoc`이 설정되어 챕터 화면으로 진입하면 페이지 상태는 그대로 보존(돌아왔을 때 같은 페이지에서 시작).

## 단위 테스트 계획
브라우저에서 dev 서버로 직접 확인:
- 문서 0개일 때: 기존 빈 상태 메시지 그대로 노출, 페이지네이션 숨김
- 문서 1~10개: 페이지네이션 컨트롤 자체가 안 보임
- 문서 11~25개: 1·2·3 페이지 버튼, 이전/다음 동작
- 문서 25개 초과 가정 시(데이터 부족하면 PAGE_SIZE를 임시 3으로 낮춰 재현): 윈도우/생략 표기(`…`) 정상
- 페이지 이동 후 문서 클릭 → 챕터 화면 → 뒤로가기 시 같은 페이지로 복귀
- 1페이지에서 "이전" disabled, 마지막 페이지에서 "다음" disabled
